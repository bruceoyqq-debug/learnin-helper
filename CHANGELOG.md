# Changelog

所有版本变更记录。

---

## [1.4.0] - 2026-03-18

### 🐛 Bug 修复

- **修复视频播完后无限循环重播**：`tryPlay` 缺少对 `video.ended` 的判断，导致视频结束后主循环仍持续触发播放。现在在 `tryPlay` 入口处增加 `if (video.ended) return` 检查，播放完成后不再重新触发。

### ⚙️ 配置调整

- 默认播放倍速由 3× 改为 **2×**

---

## [1.3.0] - 2026-03-17

### 🐛 Bug 修复

- **修复多开后新标签页不自动播放**：根本原因是新标签页无用户交互记录，插件注入时机早于视频就绪。修复方案：
  1. 检测到视频页 URL（含 `/learn/`）时自动启动刷课循环，无需手动点「开始刷课」
  2. `tryPlay` 改为持续重试机制（最多 30 次，约 30 秒），直到播放成功为止
  3. 统一改为先静音播放、成功后 500ms 恢复音量，彻底绕过浏览器自动播放策略
  4. vjs API 失败时自动降级到原生 `video.play()`

---

## [1.2.0] - 2026-03-17

### ✨ 新增

- **一键多开**：在课程列表页（`.video-items`）自动识别所有 `a.student-course-video-item` 链接，每隔 600ms 依次在新标签页打开，每个标签页独立自动播放，实现并行刷课。
- 多开按钮智能显示：仅在检测到课程列表时显示「🗂 一键多开」按钮，并实时显示可打开的视频数量；视频播放页自动隐藏该按钮。
- SPA 路由切换时自动刷新多开按钮状态。

---

## [1.1.0] - 2026-03-17

### 🐛 Bug 修复

- **修复自动播放未触发**：平台使用 video.js 播放器，直接调用原生 `video.play()` 会被拦截。现改为优先通过 video.js 的 `player.play()` API 触发；备用方案为先静音播放（绕过浏览器自动播放策略限制），播放成功后 300ms 自动恢复音量。
- **修复倍速被平台重置**：新增对 video.js `play` / `ratechange` 事件的监听，与原生事件双重绑定，防止平台在视频切换或加载时重置播放速度。

### ✨ 新增

- 自动识别并获取 video.js 播放器实例，兼容 `video.player`、`videojs.getPlayer()` 及父容器 ID 三种方式。
